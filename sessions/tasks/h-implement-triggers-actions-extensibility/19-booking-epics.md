## Summary: Trigger Events for Booking System

All booking-related events that can fire triggers:

| Event | Description | Available Context |
|-------|-------------|-------------------|
| `booking.created` | New booking created | booking_id, customer, service, staff, datetime, price |
| `booking.confirmed` | Booking confirmed | booking_id, customer, service, staff |
| `booking.cancelled` | Booking cancelled | booking_id, customer, cancelled_by, reason |
| `booking.rescheduled` | Booking time changed | booking_id, old_datetime, new_datetime |
| `booking.completed` | Appointment finished | booking_id, customer, service, staff |
| `booking.no_show` | Customer didn't show | booking_id, customer, service |
| `booking.reminder_24h` | 24h reminder time | booking_id, customer, service, datetime |
| `booking.reminder_1h` | 1h reminder time | booking_id, customer, service, datetime |
| `booking.payment_received` | Full payment made | booking_id, amount, payment_method |
| `booking.deposit_received` | Deposit payment made | booking_id, amount, payment_method |
| `booking.refund_processed` | Refund issued | booking_id, amount, reason |
| `booking.recurring_created` | Recurring series created | recurring_id, booking_ids, frequency |
| `booking.waitlist_joined` | Customer joined waitlist | waitlist_id, customer, service, date |
| `booking.waitlist_slot_opened` | Slot became available | waitlist_id, customer, slot_datetime |
| `booking.waitlist_converted` | Waitlist to booking | waitlist_id, booking_id |
| `booking.staff_assigned` | Staff assigned to booking | booking_id, staff_id |
| `booking.resource_booked` | Resource reserved | booking_id, resource_id, resource_name |
